Decorations on vPC



  1. Mypers > vpc > worklist tab > "Type" in action request box > search for "decorations" then press refresh
  2. Select aname and push "open request to finalize" under the list
  3. >PREVIEW DECORATIONS, PRESS PREV. DEC. AWARD
    [image here]
  4. Refer to this Decoration Review Checklist to ensure everything on the preview looks good:
    If good, save in the respective folder (w/ Preview)
    If not good, return to the CSS with a statement on what needs to be fixed. Then, move onto the next decoration.
  5. If good, go back on vPC and press the order number to generate order #
    Save in the respective folder for final copies
  6. Back on vPC after the push date populates, close out and move onto the next decoration.
IF THERE IS ERROR WHEN YOU PUSH, read the error. Examples of errors what it means and what you need to do:
  • ORA-20001: decor_validation: N, err_msg, Cannot input same number while adding
    This error means that the "Decoration Count" is wrong because there are already a previous decoration of the same kind before it.
    For example, let's say a member has an AFAM already, but the site (for a new AFAM) says "Decoration Count: 1", it means the Decoration Count should be 2 because the member alerady has an AFAM and this current site is pushing for a second AFAM, not a first one.
  • The document upload failed. ARMS has indicated that this document already exists for this service member during this evaluation period. You will need to remove the existing document from ARMS before this document can be uploaded.
    This means that this decoration already exists in PRDA and milPDS.
    This often happens as a glitch while pushing the decoration, where vPC will push the site twice and duplicate its request.
    In this case, save the copy of the final version of the decoration onto your desktop or folder, and then return the site back to the CSS so they can close it.
  • 196-68-1829: MilPDS Error: ORA-20001: decor_validation: N, err_msg, Cannot input decoration data on projected records or gains until the complete record has been received from HAF
    I have no idea what this means
